# GoMap v0.4.6
> secprobe 引擎收口、协议扩展增强、共享弱口令字典治理、HTTP/API 登录型认证接入

---

## 概览

本版本是相对 `v0.4.5-alpha-1` 的一次阶段性增强发布，重点围绕 **secprobe 执行架构统一**、**协议元数据收口**、**弱口令字典治理**、**新增协议支持**、**未授权模板执行器** 与 **三方集成升级路径** 展开。

这一轮更新的核心不是“再多加几个协议”这么简单，而是把 `secprobe` 的默认运行模型进一步稳定为：

`metadata -> planner -> engine -> provider`

同时，弱口令候选生成也统一收口到：

`metadata.dictionary -> credential profile -> generator -> engine`

这意味着后续协议接入、默认字典维护、结果语义、测试基线和三方库扩展，都有了更明确的一套主路径，而不是继续依赖历史 batch prober、协议特判和分散字典。

**对比基线**：`v0.4.5-alpha-1`

**更新规模**：143 commits | 192 files changed | +40,771 / -706 lines

---

## 重点内容

- **secprobe 执行模型统一**：默认主链路收口到 `metadata -> planner -> engine -> provider`，内置协议不再以 legacy batch prober 作为主心智模型
- **协议元数据目录化**：内置协议静态声明集中迁移到 `app/secprobe/protocols/*.yaml`
- **atomic credential 迁移完成**：内置 `credential` 协议批量迁移到单次原子认证模型
- **共享密码池落地**：默认弱口令从“每协议一份密码字典”切换为“全局共享密码池 + 协议差异 metadata”
- **HTTP/API 登录型认证接入**：新增 `httpauth` 复用层，并接入 `zabbix`、`neo4j`
- **协议支持面增强**：新增 `imap`、`pop3`、`ldap`、`kafka`、`activemq`、`elasticsearch` 等能力，并完成多批既有协议迁移
- **simple unauthorized template executor**：`memcached` 未授权确认改为模板执行器路径，`zookeeper` 保持 code-backed provider
- **三方接入与扩展文档完善**：补齐升级迁移指南、协议扩展指南、最小样例、流程图和阶段总结
- **CI / 集成测试稳定性增强**：修复 testcontainers `localhost / ::1` 环境差异导致的 SSH 测试不稳定

## 新功能

### secprobe 引擎与执行路径

- 新增 `metadata` 加载器、`planner`、`engine runner`、atomic registry 适配能力
- `Run` / `RunWithRegistry` 的默认执行模型进一步统一，减少 `run.go` 内的协议散写控制逻辑
- 引入 public prober compatibility adapter，兼容历史扩展，但不再作为推荐新模型

### 协议元数据与内置协议目录

- 内置协议静态信息集中迁移到 `app/secprobe/protocols/*.yaml`
- 当前内置元数据已覆盖：
  - `ftp`
  - `ssh`
  - `telnet`
  - `smtp`
  - `amqp`
  - `mysql`
  - `postgresql`
  - `redis`
  - `mssql`
  - `oracle`
  - `rdp`
  - `vnc`
  - `smb`
  - `snmp`
  - `mongodb`
  - `memcached`
  - `zookeeper`
  - `elasticsearch`
  - `imap`
  - `pop3`
  - `ldap`
  - `kafka`
  - `activemq`
  - `zabbix`
  - `neo4j`

### 新增或补齐的协议能力

- 新增 `mssql`、`rdp`、`vnc`、`smb` 的凭证探测接入
- 新增 `smtp`、`amqp`、`oracle`、`snmp` 的凭证探测与默认注册
- 新增 `mongodb` 凭证探测与默认注册
- 新增 `elasticsearch` 原子认证能力
- 新增 `imap`、`pop3`、`ldap`、`kafka` 原子认证能力
- 新增 `activemq` 原子认证能力
- 新增 `zabbix`、`neo4j` HTTP/API 登录型认证能力
- 新增 `memcached`、`zookeeper` 未授权访问探测

### HTTP/API 登录型认证复用层

- 新增 `internal/secprobe/httpauth`
- 统一封装：
  - HTTP client 构造
  - cookie jar
  - 可选 TLS 放宽
  - context / timeout 透传
  - transport 错误归类
- 当前已用于 `zabbix`、`neo4j`

### 共享弱口令字典与生成器

- 新增共享密码池：`app/secprobe/dicts/passwords/global.txt`
- 新增凭证生成器与相关基础设施：
  - 来源优先级
  - 候选去重
  - 基础变异
  - 分层过滤
  - 凭证画像
- 引入 `top / common / extended` 分层语义，不再依赖“前 N 条”这类隐式截断

### simple unauthorized template executor

- 新增模板装载与执行路径
- `memcached unauthorized` 已切换到模板执行器
- 模板当前刻意收边为：
  - 单次请求
  - 单次读取
  - 简单 matcher
  - 无循环
  - 无状态机

## 变更

### secprobe 维护方式变化

- 协议扩展模型从“代码直拼 + 运行时特判”进一步切换为“metadata 驱动静态声明 + provider 驱动单次动作”
- 默认弱口令维护方式从“每协议一份 txt”切换为：
  - 共享密码池
  - `default_users`
  - `extra_passwords`
  - `default_pairs`
  - `default_tiers`
  - `allow_empty_username`
  - `allow_empty_password`
  - `expansion_profile`

### 资源目录结构调整

- `app` 目录按 `assetprobe` / `secprobe` 拆分内置资源
- 服务探针、服务映射和目录爆破字典路径已统一调整
- 资源加载与 embed 回退测试同步补齐

### 文档体系补强

- 新增协议扩展开发指南
- 新增三方接入升级迁移指南
- 新增协议最小扩展示例
- 新增 `rsync` 边界说明
- 新增弱口令执行流程图
- 新增多阶段设计、计划、交付总结与升级说明文档

## 移除

### 旧式默认字典主路径

- 移除协议独立默认弱口令字典作为推荐主模型
- 移除 `dict_dir` / `DictDir` 作为默认 builtin 弱口令来源的主路径心智
- 默认 builtin 路径不再依赖“每协议一份 txt + 手工拼装”

### legacy 执行模型的主路径地位

- legacy public `Prober` 仍保留兼容入口
- 但不再代表当前内置协议推荐扩展方式
- builtin hot path 已切换到 atomic provider 为中心

## 修复

### 执行语义与结果一致性

- 修复分层过滤为空时被误判为 `unsupported-protocol` 的回退语义问题
- 修复显式传入凭证被自动扩展的语义偏移，保持 inline credentials 的字面语义
- 收紧档位交集、去除隐式截断，避免“看起来扫全了其实只扫了前 N 条”的误解
- 修复 legacy unauthorized 与 atomic credential 混挂时的回退回归

### 协议实现细节

- 修复 SMTP implicit TLS 路径超时与认证回退问题
- 修复 Oracle、SNMP、RDP、MongoDB 等协议的端口归一、超时透传和错误映射细节
- 修复 `zabbix`、`neo4j` 的 HTTPS scheme 与真实 HTTP 测试路径问题
- 修复 `mongodb`、`zookeeper`、`memcached` 等协议的未授权 / 鉴权确认边界

### 集成测试稳定性

- 修复 testcontainers 返回 `localhost` / `::1` 时导致的 SSH 握手不稳定问题
- 收紧 MySQL、MongoDB 等夹具就绪判定
- 默认继续跳过服务探测类集成测试，避免无 Docker 环境误报

## 说明与注意事项

- **这是一次 secprobe 主路径升级版本**：如果你只是调用 `Run` / `RunWithRegistry`，大多数 public API 不需要改，但需要接受新的内部执行模型。
- **默认弱口令维护模型已经变化**：今后不建议再为每个协议复制整份密码库，优先维护共享密码池和协议差异 metadata。
- **HTTP/API 登录型认证不是新 capability**：`httpauth` 只是 provider 层复用助手，不是 YAML DSL，也不是第二个 engine。
- **simple unauthorized template executor 是受限执行器**：只适合简单未授权确认，不适合多阶段协议状态机。
- **`rsync` 当前只完成边界评估**：尚未进入这一轮实现批次。
- **当前公开主链路固定使用 `default` 档位**：内部已有 `fast/default/full` 抽象，但尚未作为 public 参数开放。

## 升级与集成

### 作为 Go 依赖升级

如果你以库形式使用 `gomap`，建议：

```bash
go get github.com/yrighc/gomap@<new-tag>
go mod tidy
```

升级后建议重点检查：

- 是否依赖了旧的协议独立默认字典行为
- 是否依赖了 legacy public prober 的内部暴露方式
- 是否有自定义 secprobe 扩展仍直接注册 batch prober
- 是否需要把私有协议迁移到 `RegisterAtomicCredential(...)` / `RegisterAtomicUnauthorized(...)`

### 作为 CLI 使用

可以通过仓库 Releases 下载对应版本，或直接源码构建：

```bash
go build -o gomap ./cmd
```

常见验证命令：

```bash
./gomap weak -target 127.0.0.1 -ports 21,22,6379
./gomap weak -target 127.0.0.1 -ports 11211,2181 -enable-unauth
```
